# 製程規格查詢

這個技能協助查詢 WAT 與 LQC 兩類製程規格，包含目標值、規格上下限（USL/LSL）與 SPC 管制上下限（UCL/LCL）。

## 查詢前需要確認的資訊

規格查詢需要 **Tech ID**（官方技術代碼，如 D25、F46）作為必要條件，因為規格是以技術代碼為單位儲存的。如果使用者只提供了產品名稱（如 GAA110）沒有 Tech ID，先查出對應的技術代碼再進行規格查詢。查不到技術代碼時，告知使用者並請他們確認。

如果使用者有指定特定產品，記得在查詢時限定該產品——否則會把整個技術代碼下所有產品的規格都撈出來，資料量非常大且難以閱讀。

## WAT 與 LQC 的區別

- **WAT（Wafer Acceptance Test）**：晶圓驗收測試，通常在關鍵製程步驟後或 wafer 完工後進行電性量測，例如 Vth、GIDL、Array_VT 等電性參數
- **LQC（Line Quality Control）**：製程中線上品質監控，例如薄膜厚度、CD、Recess 深度等物理量測

使用者描述的參數名稱通常可以判斷屬於哪一類；如果不確定，可以兩者都查。

## 模糊查詢 vs 精確查詢

- **只有描述性文字**（如「W recess 深度」）→ 用模糊匹配查詢，系統會自動比對最接近的參數 ID
- **已知確切參數 ID**（如 `WL_W_recess`）→ 用精確查詢，速度更快、結果更準確

如果模糊查詢找不到結果，可以嘗試降低匹配信心度門檻（預設 0.5，可降至 0.3）再試一次。
