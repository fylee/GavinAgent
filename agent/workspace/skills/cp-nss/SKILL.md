# CP NSS 測試標準查詢

這個技能協助查詢各產品在特定 CP 測試站點的 NSS Yield Criteria 設定。

## 查詢前需要確認的資訊

需要確認兩件事：**產品代碼**與**測試站點**。

沒有測試站點的話，查詢結果可能非常龐大且難以解讀——因為 NSS 是 per-step 設定，同一個產品在不同站點有完全不同的標準。如果使用者只給了產品代碼卻沒說站點，先詢問他們：「請問您要查哪個測試站點（例如 S1P1、SPRE、DS03、DS04）？有特定想查的參數嗎？」

> 如果使用者明確要查「整個產品的所有 CP 標準」，可以省略站點，但要預告結果可能較多。

## Tech ID 的處理

CP NSS 查詢以**產品代碼**為單位，不接受技術代碼（如 D20、F46）。如果使用者給的是技術代碼，需要先查出該技術世代下的產品清單，再從清單中萃取共同產品前綴（例如 JAA130、JAA132 → 前綴 `JAA`；若有多個不同前綴如 JAA、TC1，則分批查詢），最後用前綴進行 NSS 查詢。

## 常見語意對應

使用者有時不會用精確術語，這裡是常見說法到查詢條件的對應：

- 「報廢標準」→ 過濾 cp6_comment 中含 `報廢` 的條件
- 「Hold 條件」→ 過濾 subcon_comment 中含 `Hold` 的條件
- 「出貨良率」→ 過濾 formula 中含 `Yield` 的條件
